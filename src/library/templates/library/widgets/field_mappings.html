Help — entry shape, auto-create, scale/offset/tags

Each row maps a decoded source field to a canonical L1 metric key.

Auto-create: typing a metric key not in the catalogue creates the L1 row on save (operators tidy label/unit in /metrics/ afterwards).

Scale / offset: apply a linear conversion value × scale + offset — used when the decoder can't emit canonical units (typical for vendor LoRaWAN codecs we don't fork). Defaults: scale 1, offset 0.

Tags: JSON like {"phase": "L1"} for multi-channel devices (3-phase meters, multi-tariff, …).
